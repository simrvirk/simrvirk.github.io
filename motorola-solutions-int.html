Overview

At Motorola Solutions' Avigilon division in Vancouver, I worked as a Design Validation Engineering Intern within the hardware team, reporting to the Mechanical Engineering group. My mandate was to build and deploy automated validation infrastructure for camera hardware — reducing manual test burden, increasing measurement precision, and generating repeatable data that feeds directly into product certification decisions.

The role sat at the intersection of mechanical systems, software automation, and computer vision. I wasn't running tests — I was designing and building the systems that run the tests. That distinction shaped everything I produced during this term.

The flagship deliverable was a fully automated Field of View (FOV) goniometer system — a Python-based, camera-driven pipeline capable of measuring angular FOV with ±0.5° accuracy across full product sweep cycles with no human intervention. It cut cycle time by 85% versus the manual baseline and is now part of the lab's standard test infrastructure.

Key Contributions

01 — Flagship Project
Automated FOV Goniometer System
Designed and built a fully automated test system for measuring camera field of view using a motorized Zaber rotary stage, RTSP video stream ingestion, and OpenCV-based image processing. Achieved ±0.5° angular accuracy and eliminated the need for manual jig alignment.
02 — Computer Vision
RTSP Pipeline & Green-Channel Detection

03 — Mechanical Correction
Nodal Point Parallax Compensation
Developed and integrated a nodal point parallax correction algorithm to compensate for lens-to-rotation-axis offset inherent in the goniometer fixture geometry. This significantly improved accuracy at wide angles where parallax error dominates.
04 — Software Architecture
Discrete Step Engine & Test Automation
Architected a modular Discrete Step Engine that sequences goniometer positioning, image capture, and data logging into a fully autonomous test cycle. Reduced full-suite cycle time by 85% versus the manual baseline and produced structured output reports consumed downstream by the validation team.
Deep Dive

FOV Goniometer
Automation System

The problem: Validating the field of view of camera products required a technician to manually position a jig, capture stills, and measure angular bounds by hand — a process that was slow, operator-dependent, and non-repeatable at the precision levels required for certification.

The approach: I designed a fully software-driven test system around a Zaber motorized rotary stage, which sweeps the camera through a calibrated angular range while the system continuously captures and analyzes live RTSP video frames. The core image processing chain uses OpenCV's green-channel thresholding to isolate a laser reference target in the scene, then tracks its centroid pixel position through each angular increment.

The architecture: The Discrete Step Engine (DSE) sequences stage movement, frame acquisition, and analysis into a closed-loop cycle. At each step, the engine: commands the Zaber stage to a target angle, waits for settle, grabs a frame from the RTSP buffer, thresholds and detects the target, logs the angle + pixel position, then advances. All timing, retry logic, and edge-case handling is encapsulated in the engine so test scripts remain clean and readable.

The challenge: Real camera lenses introduce nodal point offset — the entrance pupil is not co-located with the rotation axis of the goniometer. At wide angles this creates measurable parallax error that corrupts the FOV reading. I modelled the geometry and implemented a correction factor derived from the measured nodal point position, applied automatically at each angular step. This was the difference between a plausible demo and a production-quality validation tool.
