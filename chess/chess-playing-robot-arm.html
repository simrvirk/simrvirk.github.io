Vision Guided 5-DOF
Chess Playing Robot Arm

Engineered a 5-DOF computer-vision-guided robotic arm capable of playing adaptive Over-The-Board chess against a human opponent. A centralized Python Brain integrates an OpenCV board-state detection pipeline with the Stockfish engine — dynamically targeting the 1,650–2,000 ELO range. The core technical achievement is a custom Inverse Kinematics engine built on Homogeneous Transformation Matrices that translates AI move outputs into precise, multi-axis servo and stepper commands.

System Architecture & Software
  • Python Brain — centralized architecture handles OpenCV detection, Stockfish queries, and IK math (NumPy/SciPy) locally, then streams optimized coordinate strings over USB Serial to Arduino; eliminates MATLAB/Python split latency
  • Inverse Kinematics engine — solved 5-DOF kinematic redundancy by imposing a strict vertical end-effector constraint (gripper always perpendicular to table), reducing the matrix system to a determinate solution and guaranteeing flat piece pick/place without dragging
  • Forward Kinematics skeleton — Homogeneous Transformation Matrices calibrated to physical axes of rotation; explicitly accounts for 12.85 mm horizontal planar offsets between the bicep and forearm links introduced by the S-curve arm geometry
  • OpenCV vision pipeline — overhead camera detects board state each turn, maps piece positions to algebraic notation, and passes the result as Stockfish query input
  • Adaptive ELO targeting — Stockfish engine strength dialed to 1,650–2,000 range; user selects target rating and piece color via onboard joystick UI before each game
Hardware & Electromechanics
  • Base drive — NEMA 17 stepper + A4988 driver module on custom expansion board; 12:1 3D-printed gear reduction combined with 1/16th microstepping yields 38,400 micro-steps/rev → sub-millimeter panning accuracy under full arm load
  • Upper linkages — Shoulder, Elbow, Wrist, and Gripper driven by MG996R/SG90 PWM servos via PCA9685 16-channel I2C servo driver on Arduino Mega
  • Servo motion smoothing — custom C++ pulse-width sweep with injected microsecond delays across interpolated steps; bypasses violent default RC servo dynamics to produce smooth, industrial-grade gliding motion
  • Stepper resonance resolution — severe inertia-driven resonance and stalling under fully-assembled arm load resolved by enabling 1/16th microstepping (DIP switches) and scaling firmware to 3,200 steps/rev; eliminated vibration entirely
  • Thermal & deadband management — PWM signal actively detached during idle to prevent thermal buildup; software limits backed off by 5 pulse-widths to prevent deadband fighting against hard 3D-printed end stops
  • Closed-loop UI — 16×2 LCD, analog joystick (ELO/color selection), and base potentiometer for absolute homing reference feedback
