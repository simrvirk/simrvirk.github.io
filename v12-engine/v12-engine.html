Fully Constrained Components — Pistons, Crankshaft, Camshafts & Valve Train Across 12 Cylinders

A self-directed deep-dive into SolidWorks at the assembly level — 50+ individually modeled components constrained into a fully functional kinematic chain. The goal was never a showpiece; it was to deliberately encounter and resolve the hard problems: mate conflicts at scale, crankshaft throw geometry, design intent that survives a rebuild, and configuration management across cylinder instances.

The skills built here transferred directly. The mate-debugging instinct developed across 12 cylinders is the same instinct used in the Team Phantom inverter mount assembly. The fabrication drawing workflow practiced here was the baseline for Motorola Solutions PDM production drawings. The CSWP certification followed because the exam tests exactly what this project forced.

Scope & Assembly
  • 50+ components — pistons, connecting rods, crankshaft, camshafts, valve trains, cylinder heads, intake manifold, oil pan, spark plugs
  • 12-cylinder, 2-bank configuration — all components individually modeled with correct dimensional geometry and sketch intent
  • Fully constrained master assembly — zero suppressed conflicts; all mates resolved at subassembly level before promotion
  • Internal geometry included — piston rings seated in grooves, valve springs captured between retainer and head; accuracy over visual simplicity
  • SolidWorks Motion Study — validates full kinematic chain: crankshaft → rods → pistons, camshaft sequencing valves at half crank speed
Process & Technical Method
  • Learning by replication — followed the SOLIDWORKS Engine Design YouTube playlist feature-by-feature, rebuilding each component independently after watching; forced engagement with design intent over button-clicking
  • Feature tools exercised — lofted cuts, revolved boss/base, swept features on 3D paths, linear & circular patterns, configuration management, reference geometry planes
  • Crankshaft throw geometry — angular offsets between cylinder pairs modeled correctly; rod big-end and small-end bores dimensioned to journal and pin diameters
  • Mate conflict resolution — diagnosed cascade failures from crankshaft throw geometry interacting with rod mates across 12 cylinders; resolved methodically at subassembly level
  • Outcome — primary contributor to CSWP certification; same workflows carried directly into Team Phantom chassis design and Motorola Solutions CAD work
SolidWorks Techniques Mastered
  • Lofted cuts for crankshaft throw geometry; revolved boss/base for journals, pin bores, and piston crowns; swept features along 3D spline paths for manifold geometry
  • Design intent modeling — features built with explicit sketch intent (relations, reference geometry) rather than hard dimensions; rebuilds survive constraint edits without cascading failures
  • Configuration management — view configurations used for exploded assembly and sub-assembly isolation; design tables drove parametric control of bore and stroke across cylinder instances
  • Reference geometry — axes and offset planes used as driving references rather than model edges; geometry changes propagate without breaking downstream mates
  • Mate debugging at scale — each crankshaft throw simultaneously constrains two connecting rods; resolving cascading overdefined states across 12 cylinders built the diagnostic instinct applied in every subsequent assembly
What This Built — Direct Application
  • CSWP certification — the exam's assembly segment tests mate-conflict resolution and configuration management at depth; this project was the specific preparation that made those sections straightforward
  • Motorola Solutions — fixture assembly modeling, fabrication drawings with GD&T-compliant tolerancing, BOM documentation in SolidWorks PDM; all workflows practiced on this project before being applied professionally
  • SFU Team Phantom — inverter mount FEA in SolidWorks Simulation, sheet metal Unfold/Unbend for firewall flat patterns, multi-body assembly for chassis clearance verification; the V12 established the assembly baseline
  • The gap closed — university coursework teaches SolidWorks features; this project built the problem-solving instinct for when an assembly resists. That distinction is what separates a proficient CAD user from an engineer who designs in CAD
